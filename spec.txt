Spec Kit プロンプト集
1. /speckit.constitution (プロジェクト原則の確立)
Prompt:
2人対戦型ボードゲーム「パイプライン」の憲法を策定してください。
原則: > 1. モノトーン基調のデザインとWCAG AA（コントラスト比4.5:1以上）の厳守。
2. P2P（PeerJS）によるサーバーレス対戦。
3. 色覚多様性に配慮し、色（青・赤）だけでなく線種（実線・二重線）でもプレイヤーを識別する。
4. コードの保守性と、スマホ/PC両対応のレスポンシブな操作性を最優先する。
5. vercel単体で動くことを想定する。
6. いきなりすべての実装はせず、大枠から初めて、後に詳細な実装に入るようプランを立てる。


2. /speckit.specify (基本仕様の作成)
Prompt:
添付の「パイプライン_ルールブック.md」に基づき、詳細な仕様書を作成してください。以下の追加仕様を必ず含めてください：

得点計算: 累積加算方式。タイルを置くたびに、接続された全枚数を現在のスコアに加算する。

逆転用タイル: 有効期限は5自ターン以内。タイル上にカウントダウンを表示し、残り1ターンでパルスアニメーションによる強調を行う。

消滅と補充: 期限切れ時は自動消滅し、手札が3枚になるまで山札から即時補充する。

マッチング: PeerIDを含むURLと、それを共有するためのQRコード生成機能。URLはホスト、ゲストともに同じものを使用する。複数組が使用できるようにする( 1組目：/room/ABC 2組目：/room/ZYZ等)



3. /speckit.clarify (曖昧さの解消 - 推奨)
Prompt:
策定した仕様に基づき、実装上のリスクや曖昧な点を抽出してください。特に「P2Pでのターン同期の整合性」「ドラッグ＆ドロップ中のタイルの回転操作」「累積スコア計算の例外処理」について構造的な質問を生成してください。

4. /speckit.plan (実装計画の作成)
Prompt:
以下の技術スタックを用いたフェーズ別の実装計画を作成してください。

Next.js (App Router) / Tailwind CSS / PeerJS / @dnd-kit / qrcode.react
マイルストーン:

P2P通信基盤とQRコード共有

6x6盤面とD&Dタイル配置UI

接続判定アルゴリズム（BFS/DFS）と累積加算ロジック

逆転タイル専用UIとカウントダウン・補充ロジック

WCAG AA準拠のデザイン調整

5. /speckit.checklist (品質確認 - オプション)
Prompt:
実装計画と仕様に対し、以下のチェックリストを生成してください。

タイルが重なった際、下層のタイルがスコア計算から除外されているか。

逆転タイルのカウントダウンが相手のターンではなく「自分のターン終了時」に進んでいるか。

QRコードがスマホでスキャン可能なサイズとコントラストを維持しているか。

6. /speckit.tasks (タスクの生成)
Prompt:
実装計画を、開発者が即座に着手できる小さなアトミックなタスクに分解してください。各タスクには期待される成果物（ファイル名や関数名）を含めてください。

7. /speckit.analyze (整合性レポート - 推奨)
Prompt:
生成されたタスクが、プロジェクト憲法の「WCAG AA準拠」および「累積スコア計算ルール」と矛盾していないか分析レポートを出力してください。

8. /speckit.implement (実装の実行)
Prompt:
承認されたタスクとプランに従い、コードを生成してください。まずは、タイルのデータ構造定義と、全13種類のタイルの接続点（上・下・左・右）を定義する定数ファイルから着手してください。


運用のコツ
ファイルの添付: /speckit.specify を実行する際は、必ず最初にアップロードしてくれた 「パイプライン_ルールブック.md」 をコンテキストとして読み込ませてください。

段階的な実行: Spec Kitは一度に大きなコードを作るより、小分けにタスクをこなす方がバグが少なくなります。まずは Constitution → Specify → Plan の流れで土台を固めるのが成功の近道です。
